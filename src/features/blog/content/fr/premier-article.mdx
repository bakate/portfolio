## Bienvenue sur mon nouveau blog !

C'est passionnant de commencer cette nouvelle section. Ici, je vais partager mes réflexions et découvertes sur divers sujets techniques.

### Pourquoi MDX ?

#### Syntax highlighting

We also support syntax highlighting in MDX out-of-the-box! This example uses the default [Shiki](https://shiki.style) theme. See the [MDX integration docs](https://docs.astro.build/en/guides/integrations-guide/mdx/#syntax-highlighting) for configuration options.

```astro
---
const weSupportAstro = true;
---

<h1>Hey, what theme is that? Looks nice!</h1>
```

```tsx
const firstName = "Bakate";
const lastName = "Ba";

console.log(`Hello, ${firstName} ${lastName}!`); // Hello, Bakate Ba!
```

J'ai choisi MDX car il me permet de :

- Écrire en Markdown, ce qui est simple et rapide.
- Intégrer des composants interactifs (Astro, React, Svelte, Vue...) directement dans mes articles.
- Avoir une grande flexibilité pour le style et les animations.
